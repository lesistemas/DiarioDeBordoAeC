🗓️ 04/07 – Correção de envio de e-mail via API
  • Tarefa: Ajuste do sistema bradescard.Contestacao.API para uso da nova API de envio de e-mails, com autenticação por token.
  • Tempo gasto: 8h
  • Resumo:
    • Recebida demanda vinculada ao erro no FRONT de contestação.
    • Com apoio de Danilo, foram realizadas mudanças no código para integração da API externa de e-mails.
    • Adicionado novo parâmetro para apontar corretamente ao serviço de e-mails.
    • Como era o primeiro uso da API, foi necessário solicitar token e configurar autenticação do tipo password.
    • Nos testes, meu e-mail apresentou falha no navegador. Utilizei o e-mail do Danilo para validações.
    • Testes via Postman realizados com sucesso.
    • Alterações publicadas em homologação para testes com clientes.
  • Demanda vinculada:
    • User Story #306780 – FRONT DE CONTESTAÇÃO – API E-MAIL
    • Epic #306779 – FRONT DE CONTESTAÇÃO – API E-MAIL

  • Próximos passos: Aguardar retorno da PO e validação final dos clientes.
